Встроенные алгоритмы

Уже сейчас используя материал прошлых статей вы можете написать любую программу. Вы изучили базовые инструменты программирования на которых строится любая программа. Но если вы хотите писать большие и сложные приложения, то зачастую написание простых алгоритмов, таких как сортировка массива или возведение числа в степень становится рутинным занятием.

Скорее всего, сейчас вы не можете придумать способ отсортировать массив. Мы начнем изучать курс по алгоритмам с этой темы, но сейчас вы можете поверить, что это относительно простой алгоритм, который очень часто нужно использовать на практике.

Чтобы упростить жизнь разработчикам в языке C++ уже реализованы некоторые простые алгоритмы. Сегодня мы научимся использовать их. Пока мы не будем вникать, как именно они работают, но вы узнаете об этом из курса по алгоримам.

Комментарии
Небольшое отступление от темы. Язык C++ поддерживает использование комментариев. Это часть кода, которая будет игнорироваться компилятором. То есть вы можете прямо в коде оставлять заметки для себя, так как они никак не повлияют на работу программы. Вот пример.
#include <iostream>

using namespace std;

int main()
{
int a = 2 + 2; // Создание переменной, 2 + 2 = 4
}
Весь текст после двух слэшей не будет рассматриваться компилятором как код. Это удобно использовать, чтобы оставлять себе подсказки в коде. Теперь мы иногда будем комментировать наши примеры кода таким образом.

Вам необязательно сейчас запоминать наизусть все эти функции. Вы можете использовать наше приложение как справочник и возвращаться к таблицам в случае необходимости.

#include <iostream>
#include <cmath>

using namespace std;

int main()
{
cout.precision(2);
cout << fixed;
double a = pow(3, 3) * sqrt(2) + 2;
cout << a;
}
Программа выведет 40.18.

Указатели
Нам придется опять сделать небольшое отступление, чтобы глубже разобраться в устройстве массивов. На самом деле у каждой переменной есть своя ячейка в памяти, где хранится ее значение. А массив это несколько подряд идущих ячеек памяти. Указатель - номер ячейки памяти, где хранится переменная. Чтоб получить указатель на переменную используется символ &.
#include <iostream>

using namespace std;

int main()
{
int a = 5;
cout << &a;
}
Этот код выведет на экран номер ячейки памяти, где лежит переменная a. Также для каждого типа данных, существует тип - указатель на переменную этого типа. Простыми словами, вы можете сохранить номер ячейки памяти в переменную.
Для этого нужно лишь поставить символ * после названия типа данных.
#include <iostream>

using namespace std;

int main()
{
int a = 5;
int* b = &a;
cout << b;
}
Если у вас есть указатель на ячейку памяти, то вы можете получить значение в этой ячейке используя символ *.
#include <iostream>

using namespace std;

int main()
{
int a = 5;
int* b = &a;
cout << *b << endl; // Вывод: 5
cout << *(&a) << endl; // Вывод: 5
}
Указатели - большая и сложная часть языка C++. С опытом вы научитесь применять их, но сейчас вам нужно лишь знать о их существовании.
Вернемся к массивам. На самом деле имя массива это указатель на первую переменную в этом массиве.
#include <iostream>

using namespace std;

int main()
{
int a[5] = {1, 2, 3, 4, 5};
cout << a << endl; // Выведет адрес первой переменной в массиве
cout << *a << endl; // Вывод: 1
}
К указателю можно прибавлять числа. Например если прибавить к указателю один, то он укажет на следующую переменную в памяти. А если прибавить пять, то на переменную, которая лежит через четыре ячейки. Вспомним, что массив это несколько подряд идущих ячеек. Значит, чтоб получить переменную под индексом x, можно воспользоваться записью a[x] или *(a + x). Именно поэтому индексы начинаются с нуля, а не с единицы. Ведь прибавив ноль к указателю на первую ячейку мы получим указатель на ту же первую переменную.
Зачем это все нужно? Сейчас мы изучим встроенные функции, которые работают с массивами. Чтобы передать им массив, нужно дать им указатель на первую ячейку массива и на ячейку после последней. Давайте посмотрим пример. Функция sort() сортирует массив по возрастанию значений. Чтобы воспользоваться этой функцией нужно подключить библиотеку algorithm
#include <iostream>
#include <algorithm>

using namespace std;

int main()
{
int a[5] = {3, 1, 5, 2, 4};
sort(a, a + 5);
for(int i = 0; i < 5; i++)
{
cout << a[i] << " ";
}
}
Эта программа выведет на экран
1 2 3 4 5
Как мы видим массив оказался отсортированным. Теперь можно понять странное требование функций, работающих с массивами. Почему они просят указатель на ячейку после последней в массиве, а не на последнюю? Потому что так удобнее для разработчиков, ведь чтобы получить такой указатель нужно прибавить к названию массива размер этого массива.
Подведем итог. Если у вас есть массив a из n ячеек, то передавать его во втроенные функции нужно так.
название функции(a, a + n);
C++ очень свободный язык и вы можете передавать любой функции не массив, а просто область памяти. Для этого нужно передать указатель на первую ячейку и ячейку после последней, которая войдет в отсортированную область. Но обычно в этом нет смысла, так как расположение переменных в памяти(если они не находятся внутри массива) может вполне случайным. Иногда это может пригодиться, если вы хотите применить функцию не на весь массив, а только на какую-то его часть. Например, чтобы отсортировать все ячейки массива с индексами от 1 до 3, используется следующая конструкция.
sort(a + 1, a + 4);
Это очень важная возможность, ведь вы не всегда точно знаете размер массива. Например если вам надо считать и отсортировать массив, размер которого от одного до тысячи чисел, то вы создадите массив на тысячу элементов, но не обязательно используете все. Тогда в функцию нужно передавать именно ту часть массива, которую вы используете. Вот пример, размер массива считывается в переменной n.
int n, a[1000];
cin >> n;
for(int i = 0; i < n; i++)
{
cin >> a[i];
}
sort(a, a + n);
Обычно вы всегда знаете и держите размер своего массива в отдельной переменной, так что это не должно составить вам проблем.
Будьте аккуратнее с индексами массива и указателями. Если вы выйдете за границы памяти, которая дана вашей программе, то можете оказаться в области памяти чужой программы. Это приведет к ошибке времени исполнения и вылету вашей программы. Внимательно следите за тем, чтобы не выходить за границы массива.
Алгоритмы
В этой таблице представлены функции из библиотеки algorithm. Если функция работает с массивом, то a - указатель на начало массива, а a + n на элемент после последнего.
Название функцииЗначение
count(a, a + n, x)Возвращает количество вхождений элемента x в массив a.
copy(a, a + n, b)Копирует массив a в массив b.
fill(a, a + n, x)Заполняет массив элементами x.
replace(a, a + n, x, y)Заменяет все элементы x в массиве на элементы y.
unique(a, a + n)Оставляет в массиве только уникальные элементы. Возвращает указатель на ячейку после последней в новом массиве.
random_shuffle(a, a + n)Перемешивает элементы массива в случайном порядке.
sort(a, a + n)Сортирует массив по вохрастанию.
reverse(a, a + n)Переворачивает массив (последний элемент становится первым, предпоследний вторым и т.д).
min_element(a, a + n)Возвращает указатель на минимальный элемент в массиве.
max_element(a, a + n)Возвращает указатель на максимальный элемент в массиве.
swap(x, y)Меняет местами значения двух переменных.
min(x, y)Возвращает минимум из двух переменных.
max(x, y)Возвращает максимум из двух переменных.
