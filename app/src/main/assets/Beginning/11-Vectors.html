Vector

Массивы, которые мы изучили до этого перешли в C++ из старого языка C. Сейчас на замену им пришли контейнеры. Всего есть несколько типов контейнеров, но мы остановимся только на контейнере vector - замена стандартному массиву. Про остальные контейнеры вы сможете позже прочитать в курсе по алгоритмам.

Динамический массив

В отличии от обычного массива, vector дает возможность в любой момент изменять его размер, добавлять и удалять элементы. Чтобы использовать vector вам нужно подключить одноименную библиотеку.

#include <iostream>
#include <vector>

using namespace std;

int main()
{
vector<int> a(5);
}

В этом блоке кода мы создали vector состоящий из пяти целочисленных переменных. Давайте разберем этот синтаксис. В угловых скобках после слова vector вы указываете тип данных, который будете хранить в этом массиве. Вместе это составляет название типа данных этого массива. Значит передавая vector в функцию вы можете писать вот так.

void print(vector<int> arr)
Вернемся к прошлому примеру. После указания типа данных следует название переменной. Дальше в круглых скобках указывается количество элементов в массиве. В отличии от обычных массивов, вы можете задавать размер массива с помощью переменной. Выглядит как вызов функции, не так ли? Да, это действительно так. Это особый тип функций, называемый конструкторами. Эти функции вызываются при создании переменной такого типа, к которому привязан этот конструктор. Позже мы разберем конструкторы подробнее, сейчас вам достаточно просто знать о их существовании.

Дальнейшая работа с вектором выглядит точно также, как и работа с обычным массивом. Обращение к элементам массива выполняется через квадратные скобки.

vector<int> a = {3, 2, 1};
a[0] = 7;
Из этого примера также видно, что необязательно задавать размер массива с помощью функции. vector поддерживает объявление и через фигурные скобки. Но если вы создаете vector с помощью конструктора, то вторым аргументом можете передать значение, которым нужно заполнить массив. Например.
vector<int> a(5, 3);
for (int i = 0; i < 5; i++)
{
cout << a[i] << " ";
}
// Вывод: 3 3 3 3 3

Прямые ссылки
vector позволяет использовать сокращенную запись цикла for для прохода по своим элементам. Такие циклы называют for-each.
vector<int> a(5, 3);
for (int i : a)
{
cout << i << " ";
}
// Вывод: 3 3 3 3 3
Здесь вы через двоеточие указываете тип данных и массив, по которому будете проходить циклом. В переменную объявленную в цикле будут по очереди записываться все значения из массива. Но давайте рассмотрим следующий пример.
vector<int> a(5, 3);
for (int i : a)
{
i = 10;
}
for (int i : a)
{
cout << i << " ";
}
// Вывод: 3 3 3 3 3
После первого цикла массив a почему-то не изменился. Так происходит потому что цикл for-each копирует значение из массива в переменную i. Значит изменяя переменную i вы не можете поменять исходный массив, ведь переменная из массива и переменная в цикле это две разные переменные. Но это можно исправить. Давайте использовать указатели, которые мы изучили в прошлых занятиях.

vector<int> a(5, 3);
for (int &i : a)
{
i = 10;
}
for (int i : a)
{
cout << i << " ";
}
// Вывод: 10 10 10 10 10

Поставим амперсанд перед именем переменной в цикле. Теперь вы не копируете значения из массива в переменную. Переменная i служит ссылкой на элемент в массиве. Меняя её, меняется и массив. Также можно передавать данные и в функцию. До сих пор функция не могла менять изначальных данных, но чтобы изменить это нужно просто поставить знак амперсанда.

int make3(int &x)
{
x = 3;
}

int main()
{
int a = 5;
make3(a);
cout << a; // Вывод: 3
}

Функции для работы с vector

Вернемся к теме занятия. В отличии от массива, название vector не является указателем на первый элемент. Но что делать со встроенными функциями, которым нужны указатели? У вектора есть две функции для получения указателей - .begin() и .end(). Пример.

vector<int> v(5, 7);
sort(v.begin(), v.end());
Именно таким способом стоит передавать vector во все втроенные функции.
